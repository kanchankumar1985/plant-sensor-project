ESP32 Microcontroller

Purpose: Read sensor data and transmit via USB

Features: WiFi/Bluetooth capable, 32-bit processor

Communication: USB Serial (115200 baud)

// ESP32 Code Structure void setup() { Serial.begin(115200); Wire.begin(21, 22); // I2C pins hdc.begin(0x44, &Wire); hdc.setAutoMode(AUTO_MEASUREMENT_1MPS_LP0); } void loop() { if (hdc.readAutoTempRH(temp, hum)) { Serial.print("{\"device_id\":\"plant-esp32-01\","); Serial.print("\"temperature_c\":"); Serial.print(temp, 2); Serial.print(",\"humidity_pct\":"); Serial.print(hum, 2); Serial.println("}"); } delay(2000); }

🐳 Docker Container (TimescaleDB) - Layer 5

What is Docker? Containerization platform that packages TimescaleDB and its dependencies into a portable container.

🎯 Why Docker for Database Layer?

  • Consistency: Same database environment on any machine
  • Isolation: Database runs separately from your system
  • Easy Setup: One command installs entire TimescaleDB
  • No Conflicts: Doesn't interfere with existing PostgreSQL installations
  • Version Control: Specific TimescaleDB version guaranteed

🚀 Setup & Run Commands

1. Database Setup

docker run -d --name timescaledb -p 5433:5432 -e POSTGRES_PASSWORD=plantpass timescale/timescaledb:latest-pg14
psql -h localhost -p 5433 -U postgres -d postgres -c "CREATE DATABASE plantdb;"
psql -h localhost -p 5433 -U postgres -d plantdb -c "CREATE USER plantuser WITH PASSWORD 'plantpass';"

2. Backend Setup

cd plant-monitor/backend
python3 -m venv .venv
source .venv/bin/activate
pip install fastapi uvicorn psycopg2-binary pyserial python-dotenv

3. Frontend Setup

cd plant-monitor/frontend
npm install
npm install recharts

4. Run System

# Terminal 1: Start serial reader
cd plant-monitor/backend && source .venv/bin/activate && python3 serial_reader.py
# Terminal 2: Start API server
cd plant-monitor/backend && source .venv/bin/activate && uvicorn app:app --reload
# Terminal 3: Start React dashboard
cd plant-monitor/frontend && npm run dev
